We are building a web-based learning RPG from scratch.

---

# CORE PRINCIPLE

Everything must be driven by study material (notes).

No hardcoded gameplay logic without data backing it.

---

# STACK

* Backend: Python + FastAPI
* Frontend: simple HTML/CSS/JS (no frameworks)

---

# PROJECT STRUCTURE

Create:

/backend
main.py
combat.py
question_engine.py
codex.py

/subjects
/dsa2
questions.json
concepts.json

/static
index.html

---

# STEP 1 — CONCEPT SYSTEM

Create concepts.json:

Each concept:

* id
* name
* explanation
* difficulty

Example:
{
"id": "trees",
"name": "Trees",
"explanation": "Hierarchical structure...",
"difficulty": 2
}

---

# STEP 2 — QUESTION SYSTEM

Create questions.json:

Each question:

* id
* concept_id
* question
* options
* correct_answer
* explanation

---

# STEP 3 — QUESTION ENGINE

question_engine.py:

* load questions from JSON
* get_question(concept_id=None)
* evaluate_answer(question_id, answer)

---

# STEP 4 — COMBAT SYSTEM

combat.py:

* start_combat()
* resolve_answer()

Rules:

* correct → full damage
* wrong → zero damage + enemy bonus

Each enemy must have:

* name
* concept_id
* difficulty

---

# STEP 5 — CODEX SYSTEM

codex.py:

* log_attempt(session_id, question, result)
* get_codex(session_id)
* get_mistakes(session_id)

---

# STEP 6 — FASTAPI

main.py:

Endpoints:

POST /start_combat
POST /answer
GET /codex/{session_id}

---

# STEP 7 — FRONTEND

index.html:

* start combat button
* show question + options
* show HP bars
* show result + explanation

---

# IMPORTANT RULES

* DO NOT hardcode questions in Python
* everything must come from JSON
* keep system modular for new subjects
* keep UI simple

---

# GOAL

A working vertical slice where:

* player fights enemy
* answers questions
* learns concepts
* mistakes are tracked

---

Build cleanly, step by step.
